## 什么是小狼毫？

[小狼毫]是**Rime**输入法引擎在*Windows*上的官方发行版。它是一款非常独特、强大且深受极客和文字工作者喜爱的中文输入法。

### 核心特点：为什么小狼毫如此特别？

与**搜狗**、**微软拼音**等主流输入法相比，[小狼毫]有以下几个最核心的区别和优点：

1. 干净、安全、保护隐私

- 绝无广告：界面简洁，不会出现任何弹窗广告或推广内容。

- 不联网：所有的词库和计算都在本地完成，不会将你的输入内容上传到云端服务器，彻底杜绝隐私泄露风险。

2. 高度可定制性

[小狼毫]的所有行为几乎都可以通过修改纯文本配置文件 **（.yaml 文件）** 来定制。

你可以自定义：

- 外观：候选词横排/竖排、字体、颜色、位置等。

- 操作习惯：翻页键、中英文切换键、候选词数量等。

- 词库：轻松添加专业词库（如医学、编程、古籍等）或个人常用词库。

- 输入方案：它不仅支持拼音，还内置或可轻松安装*双拼*、*五笔*、*仓颉*、*速成*等多种输入方案。

3. “一次学习，全平台通用”

**Rime** 是一个输入引擎，在不同系统上有不同的输入法：

- Windows： **小狼毫**

- macOS： **鼠须管**

- Linux： **中州韵**

一旦你习惯了 Rime 的配置方式，你的词库和配置文件的绝大部分都可以在各个平台之间迁移，实现无缝切换。

4. 强大的智能学习能力

[小狼毫]虽然没有云词库，但拥有优秀的本地算法。它会随着你的使用，在本地生成一个属于你个人的动态词库（用户词典），记录你的用词习惯，因此会越用越顺手。

## 小狼毫适合哪些人？

- 注重隐私和安全的使用者：不希望输入习惯被记录和上传。

- 文字工作者、程序员、极客：喜欢折腾，享受完全掌控软件的感觉，并能通过定制来极致提升输入效率。

- 受够了广告骚扰的用户：只想安安静静地打字。

- 需要特殊词库或输入法的用户：例如需要输入生僻字、古文、专业术语等。

## 软件的安装

随着今天争度最新开发版的发布，也正式支持了[小狼毫]的朗读，关于[小狼毫]的无障碍开发可以看看[这篇文章]

### 安装小狼毫

目前[小狼毫]的 无障碍改进的 pr 虽然已经合并，但还没有发布新版本，我们可以安装 Nightly build [0.17.4.41.93eec2d]版本。

安装没啥可说的，正常安装，会弹出输入法设置界面，默认是没有双拼的，先那么选着。

### 导入雾凇拼音

[雾凇拼音]支持二十多种拼音方式，包括无比、ABC、双拼（搜狗、微软等）。也拥有比较不错的词库。

导入之前记得删掉`%appdata%\rime`文件夹，如果不让删除，记得结束`WeaselServer.exe`，在删除它。

#### 如果你有 git

如果你电脑安装了[git],可以使用如下命令直接 clone 它的仓库，

```cmd
git clone https://github.com/iDvel/rime-ice.git Rime --depth 1
```

如果词库更新，可以到`%appdata%\rime`直接使用下面的命令更新。

```cmd
git pull
```

#### 如果没安装 git

可以[点此下载]雾凇拼音，解压到`%appdata%\rime`即可。

#### 使用雾凇拼音

导入后点击开始菜单中的**【小狼毫】重新部署**，在找**【小狼毫】输入法设定**，勾选你想使用的拼音方案。

或在托盘图标右键小狼毫的图标，点击**重新部署**，在点击**输入法设定**即可。

### 自定义配置

刚开始导入完[雾凇拼音]后，好多热键被占用了，我们需要改动配置文件关掉它，当然如果你不像我一样设置时只勾选了一个拼音方案，有切换方案的需求的话，保留一个热键作为切换方案的热键可以去`default.yaml`里复制`hotkeys`下面的某个热键使用，反正默认的f4是肯定别用了。

并且默认的候选栏一排只有五个，我还是习惯改成九个

而且默认的大小写键是切换中英文，我给改成了切换大写。

为了符合某些输入习惯，比如在打完某句话后面直接按符号，让符号也正常上屏而不是被忽略。

还有默认是左 shift 可以切换中英文，右 shift 是没用的，我的使用习惯就像搜狗看其吧，让两个 shift 都可以切换中英文，那么怎么配置呢？

地址栏或运行输入`%appdata%\rime`新建文本文档，文件名改为`default.custom.yaml`。

复制下面的内容即可，我建议新建好这个文件在重新部署一下，它会有版本号加上的。

```yaml
patch:
  schema_list:
    - {schema: double_pinyin_mspy}
  "menu/page_size": 9
  punctuator/digit_separator_action: commit
  "switcher/hotkeys": []
  "ascii_composer/good_old_caps_lock": false
  "ascii_composer/switch_key":
    Shift_L: commit_code
    Shift_R: commit_code
```

注：上面的`  schema_list:`下面的是我的双拼方案，如果你不是双拼不要完整复制，删掉那两行即可。

同样的，想更改哪个文件就建立新的`文件名.custom.yaml`即可，然后在更改里面的配置，这样即使词库更新，你自己的配置也不会被覆盖。

具体配置文档可以[点此查看]。
